# VCP元思维系统核心概念

## 什么是元思维(Meta Thinking)

VCP的元思维系统是一套通过**超动态递归思维链**模拟结构化多阶段深度思考过程的革命性架构，使AI Agent能够进行真正意义上的分层推理与自我进化。

### 核心理念

> "将RAG从单纯的知识检索升级为思维模式检索"

传统RAG系统的局限：
- 检索的是知识片段（孤立的事实）
- 无法主动组装思考路径
- 依赖提示工程指导推理方向

VCP元思维的优势：
- 检索的是**结构化的推理框架**（元逻辑模块）
- 自动组装最适合的思考路径
- 通过思维簇组合实现动态推理

## 元思维系统三拳原理

### 第一拳：词元组捕网系统(Semantic Group Enhancement)

**作用**：将用户的自然语言输入与预设的逻辑概念网络进行匹配，生成语义更精确、意图更明确的增强查询向量。

**机制**：
- 词元组是一组具有特定逻辑关联的关键词集合
- 当用户输入触发某个词元组时，系统自动扩展检索范围
- 通过向量加权融合，增强查询向量的意图表达力

**示例**：
```
用户输入: "上周的项目进展"

匹配词元组: ["项目", "进展", "按时", "质量", "风险"]
→ 增强后的查询向量不仅包含"项目进展"，还隐含了时间、质量、风险等维度
```

### 第二拳：元逻辑模块库(Meta-Logic Chunks)

**作用**：将复杂的思考模式、推理逻辑、知识范式拆解为独立的、可复用的元逻辑模块。

**结构**：
- **思维簇(Thought Cluster)**：功能相关的元逻辑模块集合
- 每个簇是一个文件夹，包含多个.txt/.md文件
- 每个文件是一个独立的元逻辑模块

**典型思维簇**：
```
前思维簇        → 意图识别与资源预检
逻辑推理簇      → 跨域类比与第一性原理溯源
反思簇          → 一致性校验与认知整合
结果辩证簇      → 认知失调调和与拓扑重构
陈词总结梳理簇   → 知识归档与智慧传承
```

### 第三拳：超动态递归融合(Super-Dynamic Recursive Fusion)

**作用**：递归向量增强机制，每一阶段的输出都成为下一阶段输入的养料。

**机制**：
- 每个思维簇执行后，计算结果向量的平均值
- 通过加权融合更新查询向量：`newQuery = 0.8 * original + 0.2 * resultAvg`
- 实现真正的思考递进，而非简单的并行检索

## 元思考链配置

**配置文件**：`Plugin/RAGDiaryPlugin/meta_thinking_chains.json`

```json
{
  "chains": {
    "default": {
      "clusters": [
        "前思维簇",
        "逻辑推理簇",
        "反思簇",
        "结果辩证簇",
        "陈词总结梳理簇"
      ],
      "kSequence": [2, 1, 1, 1, 1]
    },
    "creative_writing": {
      "clusters": [
        "前思维簇",
        "逻辑推理簇",
        "反思簇"
      ],
      "kSequence": [3, 2, 1]
    }
  }
}
```

## 调用语法

```
[[VCP元思考:链名称::修饰符]]
```

**参数说明**：
- `链名称`：指定使用的元思考链配置（如 creative_writing）
- `::Group`：开启语义组增强
- `::Auto`：自动模式，根据查询向量自动选择最匹配的主题

**示例**：
```
[[VCP元思考]]                          # 使用默认链，不开启语义组
[[VCP元思考::Group]]                   # 使用默认链，开启语义组
[[VCP元思考::Auto::Group]]            # Auto模式，自动选主题+语义组
[[VCP元思考:creative_writing::Group]] # 指定creative_writing链+语义组
[[VCP元思考::Auto:0.7::Group]]        # Auto模式，阈值0.7+语义组
```

## 核心优势

### 1. 从知识增强到智慧增强

| 维度 | 传统RAG | VCP元思维 |
|------|---------|-----------|
| 检索对象 | 知识片段 | 结构化推理框架 |
| 思考路径 | 依赖提示 | 自动组装 |
| 递进能力 | 单阶段 | 多阶段递归 |
| 可复用性 | 低 | 高（模块化） |

### 2. 模块化设计

- 每个元逻辑模块独立存储
- 可在不同思维链中复用
- 便于知识管理和传承

### 3. 动态适应

- 根据任务类型自动选择思维链
- 根据语义组激活相关模块
- 每一阶段动态调整查询向量
