# VCP 钉钉适配器架构文档

> 本文档描述 VCP 钉钉适配器 (vcp-dingtalk-adapter) 的架构设计、模块关系和消息流转机制。

---

## 1. 架构概述

### 1.1 位置与边界

```
┌─────────────────────────────────────────────────────────────┐
│                    钉钉平台                                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ 企业内部应用 │  │ 机器人 API   │  │ 事件订阅服务 │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└───────────────────────────┬─────────────────────────────────┘
                            │ HTTPS / Stream
                            ▼
┌─────────────────────────────────────────────────────────────┐
│              VCP 钉钉适配器 (vcp-dingtalk-adapter)            │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 入站层 (Inbound)                                         ││
│  │ - Stream Receiver  - Webhook Receiver                    ││
│  └─────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 协议层 (Protocol)                                         ││
│  │ - Adapter Contract (解码/编码)                            ││
│  └─────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 客户端层 (Client)                                         ││
│  │ - VCP Channel Hub Client                                 ││
│  └─────────────────────────────────────────────────────────┘│
└───────────────────────────┬─────────────────────────────────┘
                            │ VCP Channel Hub B2 Protocol
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                   VCP 核心层                                  │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ ChannelHub  │  │ RuntimeGateway│  │ DeliveryOutbox│      │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **协议隔离** | 适配器只负责协议转换，不处理业务逻辑 |
| **双向转换** | 入站消息解码为标准事件，出站消息编码为平台格式 |
| **无状态设计** | 会话状态由 ChannelHub 管理 |
| **健康检查** | 支持健康检查和优雅关闭 |

### 2.2 核心模块

#### 2.2.1 Adapter Contract (`adapter/contract.js`)

实现标准的 Adapter 接口：

```javascript
// 入站消息解码
decodeInbound(rawPayload, context) {
  return {
    eventId: rawPayload.msgId,
    eventType: 'MESSAGE',
    session: {
      externalSessionKey: rawPayload.data.conversationId,
      bindingKey: rawPayload.data.conversationId,
    },
    payload: {
      messages: [{
        type: 'text',
        content: rawPayload.data.text.content,
      }],
    },
    metadata: {
      platformData: rawPayload,
    },
  };
}

// 出站消息编码
encodeOutbound(richReply, sessionDescriptor) {
  return {
    msgtype: 'text',
    text: { content: richReply.text },
  };
}
```

#### 2.2.2 VCP Client (`adapters/vcp/client.js`)

负责与 VCP Channel Hub 通信：

```javascript
// 发送消息到 ChannelHub
async function sendMessage({
  agentName,
  agentDisplayName,
  externalSessionKey,
  message,
  metadata,
}) {
  const response = await fetch(bridgeUrl, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Bridge-Key': bridgeKey,
    },
    body: JSON.stringify({
      eventType: 'MESSAGE',
      adapterId,
      payload: {
        session: { externalSessionKey },
        messages: [{ type: 'text', content: message }],
      },
      metadata,
    }),
  });

  return response.json();
}
```

#### 2.2.3 Stream Receiver (`adapters/dingtalk/streamReceiver.js`)

维持与钉钉的长连接：

```javascript
import { StreamService } from 'dingtalk-stream';

export async function startStreamReceiver({ onMessage, logger }) {
  const streamService = new StreamService({
    appKey: process.env.DING_APP_KEY,
    appSecret: process.env.DING_APP_SECRET,
  });

  streamService.on('message', async (event) => {
    await onMessage(event);
  });

  await streamService.start();

  return {
    close: () => streamService.stop(),
  };
}
```

## 3. 消息流转

### 3.1 入站流程 (Inbound)

```
1. 钉钉服务端推送消息
   ↓
2. StreamReceiver 接收原始事件
   ↓
3. decodeInbound() 解码为标准事件
   ↓
4. VCP Client 发送到 ChannelHub
   ↓
5. ChannelHub 路由到 RuntimeGateway
   ↓
6. RuntimeGateway 调用 AI 模型
   ↓
7. AI 生成回复
```

### 3.2 出站流程 (Outbound)

```
1. AI 生成回复内容
   ↓
2. DeliveryOutbox 投递队列
   ↓
3. encodeOutbound() 编码为钉钉格式
   ↓
4. 调用钉钉 API 发送消息
   ↓
5. 用户收到回复
```

## 4. 配置项说明

### 4.1 环境变量

| 变量名 | 说明 | 必填 |
|--------|------|------|
| `DING_APP_KEY` | 钉钉应用 AppKey | 是 |
| `DING_APP_SECRET` | 钉钉应用 AppSecret | 是 |
| `VCP_USE_CHANNEL_BRIDGE` | 是否使用 ChannelHub | 否 (默认 true) |
| `VCP_CHANNEL_BRIDGE_URL` | ChannelHub 事件端点 | 否 |
| `VCP_CHANNEL_BRIDGE_KEY` | ChannelHub 认证密钥 | 否 |
| `VCP_BASE_URL` | VCP 服务器地址 | 否 (回退用) |
| `VCP_API_KEY` | VCP API 密钥 | 否 (回退用) |

## 5. 错误处理

### 5.1 错误通知

当消息处理失败时，发送错误通知到钉钉：

```javascript
async function sendErrorNotification(rawPayload, errorMessage) {
  const errorPayload = {
    msgtype: 'text',
    text: {
      content: `⚠️ 消息处理失败：${errorMessage}`,
    },
  };

  await fetch(rawPayload.data.sessionWebhook, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(errorPayload),
  });
}
```

### 5.2 重试机制

- Stream 断线自动重连
- 发送失败指数退避重试
- 健康检查失败自动重启
